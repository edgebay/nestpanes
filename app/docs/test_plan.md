## 一、概述

### 1.1 目的

本文档定义 NestPanes 项目的测试策略、测试范围和测试用例，确保各模块功能正确性和稳定性。

### 1.2 测试框架

使用 Godot 内置的 **doctest** C++ 测试框架。

- 编译：`scons tests=yes`
- 运行：`./bin/<binary> --test --test-case-exclude="*[PCKPacker]*"`
- 测试标签格式：`[App][模块名]`

### 1.3 测试范围

| 层次       | 目录                       | 测试重点                   | 优先级 |
| ---------- | -------------------------- | -------------------------- | ------ |
| 基础设施层 | `app/app_core/tests/`      | FileSystemAccess、FileInfo | P0     |
| 业务逻辑层 | `app/app_modules/*/tests/` | FileSystem                 | P0     |
| GUI 层     | —                          | 手动测试为主               | P1     |
| 设置       | `app/settings/tests/`      | AppSettings                | P1     |

## 二、测试策略

### 2.1 测试类型

| 类型         | 范围                   | 方式             | 说明                                 |
| ------------ | ---------------------- | ---------------- | ------------------------------------ |
| **单元测试** | 基础设施层、业务逻辑层 | 自动化 (doctest) | 核心类的方法级测试                   |
| **集成测试** | 模块间交互             | 手动 + 自动化    | 模块间信号传递、数据流               |
| **UI 测试**  | GUI 层                 | 手动             | 交互事件矩阵中定义的操作验证         |
| **回归测试** | 全部                   | 自动化           | 每次构建运行单元测试，确保不引入回归 |

### 2.2 测试数据

- 测试数据目录：`app/app_core/tests/data/`
- 各模块可在自己的 `tests/data/` 下添加专用测试数据
- 测试中创建的临时文件/目录应在测试结束时清理

### 2.3 测试约定

- 测试文件命名：`test_<模块名>.h`
- 命名空间：`Test<ClassName>`
- 测试用例标签：`[App][ClassName]`
- 辅助函数使用 `static inline`
- 涉及文件系统的测试需要处理跨平台路径差异
