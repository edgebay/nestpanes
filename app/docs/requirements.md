## 一、产品概述

### 1.1 产品愿景

NestPanes 旨在成为一款现代化的 Windows 桌面应用，为日常的电脑使用提供便利，提升本地文件操作体验。

### 1.2 产品定位

NestPanes 是一款面向 Windows 平台的桌面应用，基于 Godot 4.6-stable C++ 源码构建。利用 Godot 成熟的 GUI 控件体系、场景树管理和信号系统实现高效的桌面端 UI，同时禁用不需要的 3D、物理、GDScript、网络等模块以保持精简。

### 1.3 目标用户

| 用户角色            | 描述                             | 核心诉求         |
| ------------------- | -------------------------------- | ---------------- |
| 电脑相关工作者/用户 | 工作、学习需要在电脑上进行的人员 | 提高文件管理效率 |

## 二、产品目标

### 2.1 产品目标

- **短期目标**：文件管理、多标签页和可拆分视图
- **中期目标**：设置系统完善、文本编辑、工作空间管理
- **长期目标**：插件扩展

## 四、非功能需求

### 4.1 性能需求

> 以 Godot 源码版本为准

| 指标         | 要求                                                 |
| ------------ | ---------------------------------------------------- |
| 启动时间     | 冷启动 ≤ 2 秒                                        |
| 目录加载     | 含 1000 个项目的目录 ≤ 500 ms 完成列表渲染           |
| 大目录处理   | 超大目录（>5000 项）采用分步扫描/虚拟滚动，UI 不阻塞 |
| 文件操作响应 | 剪切/复制/粘贴/删除操作 UI 响应 ≤ 100 ms             |
| 内存占用     | 空闲状态 ≤ 300 MB                                    |

### 4.2 兼容性需求

> 以 Godot 源码版本为准

| 项目     | 要求                                         |
| -------- | -------------------------------------------- |
| 操作系统 | Windows 10 及以上                            |
| 架构     | x86_64                                       |
| 显示     | 支持高 DPI 缩放                              |
| 文件系统 | NTFS / exFAT / FAT32                         |
| 路径长度 | 支持 Windows 长路径（>260 字符，需系统开启） |

### 4.3 可靠性需求

| 项目         | 要求                                 |
| ------------ | ------------------------------------ |
| 布局持久化   | 异常退出时尽量保留上次成功保存的布局 |
| 文件操作安全 | 删除操作默认移至回收站，避免数据丢失 |
| 错误处理     | 文件操作失败时给出明确的错误提示     |

### 4.4 可维护性需求

| 项目     | 要求                                           |
| -------- | ---------------------------------------------- |
| 模块化   | 功能模块独立组织，可独立开发和测试             |
| 测试覆盖 | 核心模块（文件系统操作、布局管理）需有单元测试 |
| 代码规范 | 遵循 Godot Engine C++ 编码规范                 |

## 五、用户流程

> 流程图，支持多种不同的操作方式

### 5.1 文件浏览流程

1. 用户启动应用，显示欢迎页或恢复上次布局
2. 在导航窗格（左侧树状视图）展开驱动器/文件夹
3. 单击文件夹，文件窗格（右侧列表视图）显示该目录内容
4. 双击文件，根据文件类型在新标签页中打开或使用系统默认应用打开

### 5.2 文件操作流程

1. 在文件窗格或导航窗格中选中一个或多个文件/文件夹
2. 通过右键菜单或快捷键执行操作（剪切/复制/粘贴/删除/重命名）
3. 操作结果即时反映在文件列表中
4. 若操作失败，显示错误提示
5. 用户可通过 Ctrl+Z/Ctrl+Y 撤销/恢复操作

### 5.3 布局管理流程

1. 用户通过 Ctrl+T 新建标签页
2. 用户拖拽标签页到窗格边缘进行拆分
3. 用户通过 Ctrl+B / Ctrl+Shift+B 切换侧边栏
4. 应用退出时自动保存当前布局
5. 下次启动时自动恢复布局

### 5.4 导航流程

1. 在地址栏中点击面包屑按钮快速跳转到上级目录
2. 点击地址栏文本区域进入编辑模式，输入路径后回车导航
3. 使用工具栏前进/后退按钮在历史路径间切换
4. 使用上一级按钮返回父目录

## 六、数据需求

### 6.1 数据存储

| 数据类型 | 存储方式 | 位置                   | 说明                       |
| -------- | -------- | ---------------------- | -------------------------- |
| 应用设置 | 配置文件 | `%APPDATA%/NestPanes/` | 用户配置项（主题、语言等） |
| 布局数据 | 配置文件 | `%APPDATA%/NestPanes/` | 窗口布局状态、标签页信息   |

### 6.2 数据安全

- 文件删除操作默认使用系统回收站，不直接删除
- 配置文件损坏时应能回退到默认配置
